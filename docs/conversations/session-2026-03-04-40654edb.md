> [!info] **Assistant**
> This is a **Tauri 2.0** app with a **React + Vite** frontend. Here's how to run it on each platform:
> 
> **Web (browser only)**
> 
> The frontend is a standard Vite + React app, so you can run it directly in the browser:
> 
> ```bash
> bun run dev
> ```
> 
> This starts the Vite dev server at `http://localhost:1420`. Note that any Tauri-specific features (like `@tauri-apps/api` calls for PTY/terminal, dialogs, etc.) won't work in the browser — you'll need to guard those behind platform checks like:
> 
> ```ts
> const isTauri = '__TAURI__' in window;
> ```
> 
> **Desktop (macOS — current setup)**
> 
> ```bash
> make dev
> **or directly:**
> cargo tauri dev
> ```
> 
> For a production build:
> 
> ```bash
> make build
> **or:**
> cargo tauri build
> ```
> 
> **iOS App**
> 
> Tauri 2.0 has built-in iOS support. To set it up:
> 
> **1. Prerequisites**
> - Xcode installed (with iOS simulator)
> - `rustup target add aarch64-apple-ios aarch64-apple-ios-sim`
> 
> **2. Initialize iOS target**
> ```bash
> cargo tauri ios init
> ```
> This creates a `src-tauri/gen/apple` directory with the Xcode project.
> 
> **3. Run on simulator**
> ```bash
> cargo tauri ios dev
> ```
> 
> **4. Run on a physical device**
> ```bash
> cargo tauri ios dev --device
> ```
> (Requires an Apple Developer account and signing configured in Xcode.)
> 
> **5. Build for release**
> ```bash
> cargo tauri ios build
> ```
> 
> **Important caveat for iOS**
> 
> Your app uses `portable-pty` for terminal emulation, which spawns OS processes. **This will not work on iOS** — Apple's sandbox prohibits spawning child processes. You'll need to either:
> 
> - Disable/stub the PTY functionality on iOS
> - Replace it with a remote terminal connection (e.g., SSH to a server)
> - Use a WebSocket-based terminal backend
> 
> The dockview layout, React UI, and Zustand state will all work fine on iOS — it's just the native PTY that needs a platform-specific alternative.
