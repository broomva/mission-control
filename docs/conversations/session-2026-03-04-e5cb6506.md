> **Three-Column Layout + Terminal Session Persistence**
> 
> **Context**
> 
> Mission Control needs a professional IDE-like layout where selecting a project opens a full workspace: terminals on the center, file tree / git graph on the right. Critically, **terminal sessions must persist across app restarts and reboots** — when a user returns, they see their terminal tabs with scrollback history intact, ready to continue orchestrating coding agents.
> 
> Current state: flat sidebar + dockview with ephemeral terminals (lost on restart). No scrollback persistence. File tree is a dockview panel instead of a dedicated sidebar.
> 
> **Architecture Overview**
> 
> ```
> AppShell
> ├── Toolbar (44px, glass, drag region)
> └── AppBody (flex row)
>     ├── Sidebar (200px) — project list, active indicator
>     ├── DockviewContainer (flex: 1) — terminals + dashboard
>     └── ContextPanel (280px, conditional) — right sidebar
>         ├── Tab bar: Files | Git
>         ├── FileTreeView (when Files active)
>         └── GitGraphPlaceholder (when Git active)
> ```
> 
> **When no project selected:** Two-column (sidebar + dockview with dashboard).
> **When project selected:** Three-column. Center shows project terminals, right shows context.
>
