> **M3.1: Replace NDJSON/TUI Parser with Claude Code Hooks**
> 
> **Context**
> 
> M3 shipped agent spawning, PTY management, xterm.js rendering, and a timeline sidebar. But the parser that feeds the timeline is fragile — it uses heuristic string matching on Claude Code's interactive TUI output (detecting patterns like "Reading", "Flowing", "$ command"). Any Claude Code update can break it, and it produces no data in many cases.
> 
> **Claude Code hooks** are the official API: they fire structured JSON events at 18 lifecycle points (SessionStart, PreToolUse, PostToolUse, Stop, etc.) via HTTP POST. This replaces all string parsing with reliable, structured event delivery.
> 
> **Architecture**
> 
> ```
> ┌────────────────────────────────────────────────────────┐
> │  Tauri App                                             │
> │                                                        │
> │  ┌──────────────┐     ┌──────────────────────────┐     │
> │  │ AgentService │     │ HookServer (axum)        │     │
> │  │              │     │ 127.0.0.1:{random_port}  │     │
> │  │ spawn():     │     │                          │     │
> │  │  1. Write    │     │ POST /hooks/session-start │────→ AgentStatusEvent
> │  │     hooks    │     │ POST /hooks/pre-tool-use  │────→ AgentStatusEvent
> │  │     config   │     │ POST /hooks/post-tool-use │────→ AgentStatusEvent
> │  │  2. Open PTY │     │ POST /hooks/stop          │────→ AgentStatusEvent
> │  │  3. Reader   │     │ POST /hooks/session-end   │────→ AgentStatusEvent
> │  │     thread   │     │ ...                       │     │
> │  └──────┬───────┘     └──────────────────────────┘     │
> │         │                        ↑                     │
> │    Raw PTY bytes           HTTP POST (hooks)           │
> │         │                        │                     │
